UX Improvement Approaches

Two areas, three strategies — pick the approach that fits your priorities

Connections Auth Wizard

A

Incremental Fix

Keep the current step wizard but add: retry on each step, inline health check after auth completes, and a bulk scope importer (paste comma-separated IDs). Smallest change, fastest to ship.

Pros

  • Minimal code change
  • Ships in days
  • Low regression risk

Cons

  • Still feels like a basic wizard
  • No error recovery for expired sessions
  • Scope editor still separate page
B

Unified Connection Sheet

Replace the multi-page wizard with a single sliding sheet (panel from right). Platform selection → auth → health check → scope config all in one flow. Each step animates in-place. Error? Retry inline without losing context. Scope editor gets a searchable group/user picker that pulls from the connected platform.

Pros

  • Feels modern (Linear/Vercel vibe)
  • No page navigation during auth
  • Integrated health + scope in same flow
  • Retry without restart

Cons

  • Medium effort (~3-5 days)
  • Needs new scope search API
  • Sheet layout needs responsive handling
C

Connection Hub

Full redesign — a dedicated "Connections" hub page with real-time status cards, inline auth modals, health monitoring dashboard, and a visual scope manager showing actual group names/avatars. Think Vercel's integrations page.

Pros

  • Best possible UX
  • Real-time health visibility
  • Visual scope management
  • Professional feel

Cons

  • Large effort (~1-2 weeks)
  • Needs new APIs (group name resolution, avatars)
  • Risk of scope creep

Flow Builder

D

Refactor & Fix

Break the 1400-line editor into composable components. Add variable autocomplete ({{trigger.*}}), unsaved-changes indicator, and field-level validation. Keep the same layout. Fix the debugger to use WebSocket instead of polling.

Pros

  • Addresses core pain points
  • Maintainable codebase after
  • No visual redesign needed

Cons

  • Autocomplete needs variable schema work
  • Still same basic node palette UX
  • Debugger still overlay, not integrated
E

Smart Editor

Refactor into components PLUS: intelligent node palette with fuzzy search + suggested nodes based on context, rich variable autocomplete with type hints in a dropdown, inline validation with fix suggestions, and a docked execution panel (not overlay) with real-time WebSocket updates. Unsaved changes tracked with auto-save draft.

Pros

  • Major productivity boost
  • Feels like a real IDE
  • Context-aware suggestions
  • Auto-save prevents data loss

Cons

  • Medium-large effort (~1 week)
  • Variable type system needs design
  • Auto-save needs conflict resolution
F

Flow Builder 2.0

Complete rebuild with: command palette (⌘K) for all actions, node templates/snippets, multi-tab editing, split-pane debugger with variable inspection, flow diff/versioning UI, collaborative indicators. Think VS Code for flows.

Pros

  • Best-in-class flow editing
  • Scales to complex flows
  • Power-user features

Cons

  • Large effort (~2-3 weeks)
  • Complex state management
  • Risk of over-engineering

MY RECOMMENDATION

B + E — Unified Connection Sheet + Smart Editor. Best balance of impact vs effort. Delivers a noticeably better experience without over-engineering. The refactor work in E also makes future improvements (F-level features) much easier to add later.
